Klayons Authentication API provides secure user registration and login with OTP verification via email.


üîë Features:

  • User registration with comprehensive profile fields
  • Email OTP verification for registration and login
  • Token-based authentication
  • Secure password validation
üìù Registration Flow
Step 1: Register user with details ‚Üí Receive OTP via email
Step 2: Verify OTP ‚Üí Get authentication token
1. Register User
POST
{{ base_url }}/api/auth/register/
Request Body: 
{
    "username": "john_doe",
    "email": "john@example.com",
    "phone_number": "1234567890",
    "society_name": "Green Valley",
    "flat_no": "A-101",
    "password": "secure_password123",
    "confirm_password": "secure_password123"
}
Response: 
{
    "message": "Registration successful. OTP sent to your email.",
    "email": "john@example.com"
}
2. Verify Registration OTP
POST
{{ base_url }}/api/auth/verify-otp/
Request Body: 
{
    "email": "john@example.com",
    "otp_code": "123456",
    "purpose": "registration"
}
Response: 
{
    "message": "Email verified successfully! Registration complete.",
    "token": "9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b",
    "user": {
        "id": 1,
        "username": "john_doe",
        "email": "john@example.com",
        "phone_number": "1234567890",
        "society_name": "Green Valley",
        "flat_no": "A-101"
    }
}

üìù Important Notes
  • OTP Development: Currently using console email backend - OTPs will appear in server logs
  • Token Authentication: Save the token and use it in Authorization header for protected endpoints
  • OTP Validity: OTP codes expire after 10 minutes
  • Password Requirements: Minimum 8 characters with Django's default validators
  • Content-Type: Always use application/json for POST requests
